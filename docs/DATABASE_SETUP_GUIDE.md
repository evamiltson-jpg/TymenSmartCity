# Инструкция по настройке базы данных

## Для полного восстановления сервисов и проектов выполните следующие шаги:

### 1. Откройте Supabase Dashboard
- Перейдите на https://app.supabase.com
- Выберите проект `goklwsqotbwsyzctzknh`

### 2. Откройте SQL Editor
- На панели слева найдите **SQL Editor** (значок {}  )
- Нажмите **New Query**

### 3. Скопируйте и выполните SQL скрипт
- Откройте файл `SUPABASE_SETUP.sql` в этом репозитории
- Выделите весь контент SQL файла и скопируйте его
- Вставьте его в SQL Editor в Supabase

### 4. Выполните запрос
- Нажмите кнопку **Run** (или `Ctrl+Enter`)
- Дождитесь завершения выполнения
- Если всё прошло успешно, вы должны увидеть зелёные галочки

### 5. Проверьте результат
- Перейдите в **Table Editor** (левая панель)
- Вы должны видеть новые таблицы:
  - `services` (10 сервисов Тюмени)
  - `public_projects` (10 примеров проектов)

Все изображения загружаются из **Unsplash API** - безлимитного источника красивых фотографий:

## Если что-то пошло не так

### Ошибка: "Table already exists"
Это нормально - SQL скрипт использует `CREATE TABLE IF NOT EXISTS`, поэтому если таблицы уже есть, они не будут пересозданы. Если нужно пересоздать с нуля:

1. Перейдите в **Table Editor**
2. Выберите таблицу `services`
3. Нажмите **...** → **Delete table**
4. Повторите для `public_projects`
5. Снова выполните SQL скрипт

### Ошибка: "Permission denied"
Это означает, что RLS политики не установлены корректно:
1. Убедитесь что вы выполняете скрипт как admin
2. Или запустите отдельно только CREATE TABLE и INSERT части

### Изображения не загружаются
Это может быть из-за блокировки Unsplash в вашем регионе:
1. Проверьте интернет-соединение
2. Попробуйте открыть https://images.unsplash.com/ напрямую
3. Если не работает, замените URL на другой источник (Pixabay, Pexels)
