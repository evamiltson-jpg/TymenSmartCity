# Настройка создания проектов

## 1. Обновите базу Supabase

1. Откройте [Supabase Dashboard](https://supabase.com) → ваш проект → **SQL Editor**
2. **Если профили и команды уже настроены** — выполните только файл `docs/PROJECTS_MIGRATION.sql` (безопаснее, без пересоздания старых таблиц). Скрипт сам удалит старое представление `projects` (view), если оно мешает
3. **Если база пустая** — выполните весь `docs/SUPABASE_SETUP.sql` целиком
4. Если таблица `projects` уже была в старом виде — скрипт добавит недостающие колонки автоматически

> Не используйте `CREATE TYPE IF NOT EXISTS` — в PostgreSQL для ENUM это не работает. В актуальном скрипте это уже исправлено.

## 2. Проверьте Storage

В **Storage** должен появиться публичный bucket `project-images` (до 3 МБ, JPG/PNG/WebP).

## 3. Единая таблица `projects`

Все проекты (и ваши идеи, и примеры с витрины) хранятся в **одной** таблице `projects`.

| Поле | Назначение |
|------|------------|
| `moderation_status` | `На модерации` → `Принят` / `Отклонён` / `Удалён` |
| `is_on_site` | `true` — виден на сайте (портфолио, поиск), `false` — только в ЛК |
| `published_at` | время автопубликации (~1 ч после создания) |

| Этап | moderation_status | is_on_site | Где видно |
|------|-------------------|------------|-----------|
| Создание | На модерации | false | ЛК |
| Через ~1 ч (авто) | Принят | true | ЛК + сайт |
| Отклонён | Отклонён | false | ЛК |
| Удалён пользователем | Удалён | false | нигде |

Выполните также `docs/PROJECTS_UNIFIED.sql` (или повторно `docs/PROJECTS_SQL_SAFE_RERUN.sql`) — примеры на сайт и настройка RLS. Таблица `public_projects` не нужна.

## 4. Новые поля + модерация (обязательно)

Выполните в SQL Editor:

`docs/PROJECTS_COLUMNS_PATCH.sql`

Это добавит модерацию (`На модерации` → публикация через 1 час) и защиту от дублей.

Если проекты уже продублировались, один раз выполните:

`docs/PROJECTS_DEDUPE.sql`

## 5. Унификация (обязательно после п.3–4)

`docs/PROJECTS_UNIFIED.sql`

Если скрипт уже запускали и видите `policy ... already exists` — выполните вместо него:

`docs/PROJECTS_SQL_SAFE_RERUN.sql` (можно запускать повторно)

**Порядок SQL только один раз с нуля:**

1. `PROJECTS_MIGRATION.sql`
2. `PROJECTS_COLUMNS_PATCH.sql`
3. `PROJECTS_UNIFIED.sql` (или `PROJECTS_SQL_SAFE_RERUN.sql` при повторе)

Проверка: `SUPABASE_PROJECTS_CHECK.sql`

## 6. Как пользоваться формой

1. Войдите в аккаунт (создание проекта доступно только авторизованным)
2. Перейдите на страницу **Проекты** → вкладка **Создание проектов**
3. Заполните поля по шаблону, при необходимости:
   - выберите команду из списка (команды создаются в профиле)
   - загрузите обложку проекта
   - откройте калькулятор экономического эффекта (кнопка 🧮)
   - отключите рейтинг (проект будет ниже в ленте)

## 7. Новости вузов

Новости ТИУ и ТюмГУ обновляются автоматически через GitHub Actions **2 раза в сутки** (05:00 и 17:00 UTC).

Локально можно обновить вручную:

```bash
npm run parse:university-news
```

На сайте новости подгружаются при открытии страницы и каждые 30 минут, а также при возврате на вкладку.

## 8. Если «Failed to fetch» / ERR_CONNECTION_RESET

1. Проверьте интернет и отключите VPN/блокировщики на время теста
2. Убедитесь, что проект Supabase **не приостановлен** (Dashboard → Project status)
3. Создайте проект **без обложки** — сначала сохраняется текст, картинка грузится отдельно
4. В `.env.local` должны быть корректные `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`

## 9. Деплой

После `git push` на GitHub убедитесь, что в репозитории включены **Actions** (Settings → Actions → Allow all actions).
