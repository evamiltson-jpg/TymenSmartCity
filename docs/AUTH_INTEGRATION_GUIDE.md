## 🔧 Как использовать

### Первичная настройка (ВАЖНО!)

1. **Выполнить SQL скрипт в Supabase:**
   - Откройте [https://supabase.com](https://supabase.com)
   - Перейдите в **SQL Editor**
   - Скопируйте содержимое файла `SUPABASE_SETUP.sql`
   - Выполните запрос

2. **Настроить Email OTP:**
   - Перейдите в **Authentication** → **Providers** → **Email**
   - Включите **Enable Email Provider** и **Enable OTP**
   - Установите **OTP Expiry** = 15 минут

3. **Настроить шаблон письма:**
   - В **Email Templates** найдите **Confirm signup email**
   - Установите текст (см. `SUPABASE_AUTH_SETUP.md`)
   - Установите "From Email Name" = "Smart City - Тюмень"

4. **Запустить приложение:**
   ```bash
   npm run dev
   ```

## 🎯 Использование в приложении

### Для пользователя:
1. Нажимает кнопку **"Вход"** в хедере
2. Вводит email
3. Получает 6-значный код на почту
4. Вводит код в приложение
5. Автоматически перенаправляется в профиль
6. Заполняет информацию профиля (ФИО, место работы, навыки)
7. Может просматривать заявки, проекты и команды

### Для администратора:
- Управлять статусами заявок через админ-панель (доп. компонент)
- Отслеживать активность пользователей

## 📋 Функции профиля

### Вкладка "Профиль"
- Аватар (инициалы)
- Отображение основной информации
- Кнопка "Редактировать" для заполнения:
  - ФИО
  - Место работы/учебы
  - Специальность
  - Навыки (добавляются через запятую)

### Вкладка "Заявки"
- Отображение всех заявок пользователя
- Статусы: На рассмотрении, Одобрена, Отклонена
- Дата подачи заявки

### Вкладка "Мои проекты"
- Список проектов, в которых участвует пользователь
- Роль в проекте
- Статус: Активный/Завершен

### Вкладка "Мои команды"
- Список команд, в которых состоит пользователь
- Количество членов
- Кнопка для перейти к команде

## ⚠️ Важные замечания

1. **OTP коды** - вводятся как 6 цифр, отправляются на email
2. **Письма от "Smart City - Тюмень"** - подпись настраивается в Email Settings
3. **Профиль создается автоматически** при первом успешном входе

## 🆘 Если что-то не работает

1. Проверьте консоль браузера (F12) на ошибки
2. Посмотрите логи в Supabase → Logs
3. Убедитесь, что Email Provider включен
4. Проверьте, что таблицы созданы (SQL Editor → Tables)
5. Проверьте, что RLS политики включены
6. Убедитесь, что вы используете правильные URL и ключи Supabase
