# 🚀 Быстрый старт: Система авторизации Smart City Tyumen

## ⚡ 5 минут для включения системы авторизации

### Шаг 1: Выполнить SQL скрипт (2 мин)
```bash
1. Откройте https://supabase.com и войдите в проект
2. Перейдите в SQL Editor
3. Скопируйте содержимое файла SUPABASE_SETUP.sql
4. Нажмите "Run" (должно быть "Success")
```

### Шаг 2: Включить Email OTP (2 мин)
```bash
1. Authentication → Providers → Email
2. Toggle "Enable Email Provider" = ON
3. Toggle "Enable OTP" = ON
4. OTP Expiry = 900 (15 минут)
5. Сохраните изменения
```

### Шаг 3: Настроить письма (1 мин)
```bash
1. Authentication → Email Templates → Confirm signup email
2. Установите текст из SUPABASE_AUTH_SETUP.md
3. Authentication → Email Settings
4. From Email Name = "Smart City - Тюмень"
5. Сохраните
```

### Готово! ✅
```bash
npm run dev
# Сайт доступен на http://localhost:5173
```

## 🎮 Тестирование

1. Нажмите кнопку **"Вход"** в правом углу хедера
2. Введите email (любой)
3. Проверьте входящую почту на код (6 цифр)
4. Введите код в приложение
5. Вы автоматически перейдете в профиль
6. Заполните информацию:
   - ФИО: Ваше имя
   - Место работы/учебы: Организация
   - Специальность: Ваша должность
   - Навыки: React, TypeScript (через запятую)
7. Нажмите "Сохранить"

## 📧 Отправка писем

### В разработке (dev):
- Письма отправляются через Supabase
- Некоторые письма могут попадать в спам
- Проверьте логи: Authentication → Logs

### В production:
- Используйте SendGrid, Resend или другой провайдер
- Установите в Supabase Email Settings
- Проверьте домен (SPF, DKIM)

## 🐛 Если что-то не работает

### Письма не приходят?
```bash
1. Проверьте папку "Спам" в почте
2. Используйте другой email
3. Проверьте логи в Supabase → Logs → Auth
4. Убедитесь, что Email Provider включен
```

### Не могу войти?
```bash
1. Проверьте консоль браузера (F12)
2. Убедитесь, что таблицы созданы (SQL Editor → Tables)
3. Проверьте RLS политики: Authentication → Policies
4. Убедитесь, что OTP включен
```

### "Таблица не существует"?
```bash
1. Откройте SQL Editor
2. Выполните скрипт SUPABASE_SETUP.sql заново
3. Проверьте отсутствие ошибок (должно быть "Success")
4. Обновите страницу браузера
```
